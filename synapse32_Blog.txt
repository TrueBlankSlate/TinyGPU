Hello Everyone. In today’s blog I will be trying to explain the synapse 32 architecture.



Synapse 32 is an  32 bit RISC V CPU coded  using verilog .It functions as a linker between hardware and software.   


Firstly we will talk about how the code flows and through what stages does it go. This process is called pipeline and it has 5 main parts
    • Fetch Instructions code
    • Decode code
    • Execute code
    • Register memory
    • Writeback



1.] FETCH INSTRUCTIONS CODE


This process uses  the parts:-PC, IMEM.
    • PC:-   Program Counter is firstly used to store the address of the instructions. I would like to stress on the fact that it stores the address of instructions and not the  instruction itself. The inputs of a PC is given by Control unit
    • i_mem:-It acts like a storage for storing instructions. Its job is to receive input from program counter(address of the instruction) and give the instruction stored in I_Mem to the decoder. 2]Decode:-
This part uses :- decoder,control unit,register file

Now these memory types go to the decoder. Here the 32 bits from the above step are sliced by the decoder .The code for slicing may be done in this manner:-

opcode = instruction[6:0];
rd      = instruction[11:7];
funct3  = instruction[14:12];
rs1     = instruction[19:15];
rs2     = instruction[24:20];
funct7  = instruction[31:25];
Then this part is given to the register file which provides the value of the instructions and send it the control unit. Now control unit reads the data, fetches the data from register file using the address information  and send that data over to the ALU for execution.

This part uses ALU:-

 4]Register Memory
This part uses CU,Register memory,D_MEM.


In this step the data that the CU got from ALU is stored in Register file and D_MEM (Data Memory). This data may be the value of a variable and its address(rd variable in the fetch section).

    • Register file :-  It is an ultra fast data and memory storage  that is suitable for small things like 16/32 bit  number storage. Not suitable for storing larger numbers
    • D_MEM:-   Its another memory that is a bit slower than Register file ,however it can store larger inputs like vectors,arrays, strings , objects, etc. If the memory of Register file is full then the CPU may input its data in D_MEM .





5] Writeback

This stage sends  whatever data we have in Register file and D_MEM to the output. This output may be something like a seven segment display,etc.
